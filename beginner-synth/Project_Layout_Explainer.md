# Beginner Synth Project Layout (Quick Explainer)

This file explains what was created so far and why it exists.

## Top-level workspace

- `JUCE/`
  - JUCE framework source code you cloned from GitHub.
  - Your plugin project links to this directory during CMake configure/build.

- `beginner-synth/`
  - Your actual plugin project.

- `VST_Modular_Synth_Learning_Guide.md`
  - High-level learning roadmap.

- `VST_Day1_Coding_Worksheet.md`
  - Step-by-step Day 1 implementation worksheet.

---

## Inside `beginner-synth/`

- `CMakeLists.txt`
  - Build configuration for this plugin project.
  - Declares plugin name, VST3 format, JUCE dependency, source files, and compile settings.

- `Source/`
  - Contains your plugin C++ code.

- `build/`
  - Auto-generated by CMake configure.
  - This is where generated build files, object files, intermediate artifacts, and final plugin outputs are placed.

---

## Inside `beginner-synth/Source/`

- `PluginProcessor.h`
  - Main plugin processor class declaration.
  - Declares audio/MIDI lifecycle methods such as `prepareToPlay` and `processBlock`.

- `PluginProcessor.cpp`
  - Main plugin processor implementation.
  - Handles audio processing and MIDI handling logic.
  - Right now it is a minimal scaffold (mostly silence) ready for your DSP code.

- `PluginEditor.h`
  - Plugin UI class declaration.

- `PluginEditor.cpp`
  - Plugin UI implementation.
  - Currently a basic placeholder editor window.

---

## Inside `beginner-synth/build/` (from configure)

These are generated files; you generally do not edit them manually.

- `CMakeCache.txt`
  - Cached configuration values chosen during CMake configure.

- `build.ninja`
  - Ninja build graph generated by CMake.

- `CMakeFiles/`
  - CMake internal metadata and compiler checks.

- `JUCE/`
  - Generated build integration files for JUCE.

- `BeginnerSynth_artefacts/`
  - Intermediate/packaging output area for the plugin target.

---

## What happens next

1. Configure (already done):
   - Generates `build/` and toolchain/build system files.

2. Build (next command you will run):
   - Compiles C++ sources and links plugin binaries.

3. Locate plugin output:
   - After build, we will find the `.vst3` bundle together and set your host scan path if needed.

---

## Beginner rule of thumb

- Edit only:
  - `CMakeLists.txt` (project settings)
  - files in `Source/` (your code)

- Do not hand-edit:
  - anything in `build/` (generated files)
